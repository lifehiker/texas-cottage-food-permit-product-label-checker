### Email (only if PRD explicitly requires it)
- Use Resend only when the PRD requires transactional email
- **CRITICAL: NEVER instantiate `new Resend(...)` at module level** â€” always inside the handler function body with a guard:
  ```ts
  // WRONG â€” causes build failure (next build evaluates module-level code):
  const resend = new Resend(process.env.RESEND_API_KEY);
  export async function POST(req) { ... }

  // CORRECT â€” lazy init inside handler:
  export async function POST(req) {
    if (!process.env.RESEND_API_KEY) {
      console.log("[email] RESEND_API_KEY not set, skipping email");
      return NextResponse.json({ ok: true });
    }
    const { Resend } = await import("resend");
    const resend = new Resend(process.env.RESEND_API_KEY);
    ...
  }
  ```
- Next.js evaluates module-level code during `next build` page data collection â€” missing env vars at module scope crash the build
- Same rule applies to Stripe, OpenAI, and ALL third-party SDK clients â€” always lazy-initialize inside handler functions

- **Do NOT use `next/font/google`** â€” it fetches font files during `next build`, which can hang or fail in automated/offline build environments.

- If a custom font is truly required, commit the font file into the repo and use `next/font/local`; never depend on a build-time network font fetch.

- Coolify with Docker â€” always set `output: "standalone"` in next.config.ts
- Do NOT add `COPY public ./public` in the Dockerfile if the public/ directory is empty or doesn't exist
